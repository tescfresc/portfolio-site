TF'S ZOMBIE OPTIONS
A MOD FOR BLACK OPS 3
THE MOD
This is a mod for Call of Duty: Black Ops 3 which lets you pick and set different options before each zombies game. This mod came about as a need to be able to control certain aspects of the game, similar to how future sequels have allowed, in a way that felt like it was part of the base game. Most similar mods up to this point used a menu that appeared once a game was started, and had to use detection of game events, such as meleeing or firing, to control navigation. This was due to a limitation in the mod tools. The mod can be found here or through the steam workshop. This mod was one of the first mods to edit the main menu of the game, allowing for a more immersive and fitting experience.
HOW ITS MADE
The game side of the mod was made using the official Black Ops 3 mod tools. By default, the tools do no include any way to save any data through game restarts. To combat this, a member of the community, known as Jarik, made a tool that uses the game's ability to save attachments as a way of saving numbers from 0-255 in 700+ different slots. By using 2 slots to save each number, and a bit of tweaking to the script, it was possible to save values up to around 65,000. I used this to allow the mod to save your chosen options through game restarts and also to get the menu to actually work once the game was started. Black Ops (and probably other Call of Duty games) uses a form of LUA for its menu systems, known as LUI. The mod tools don't officially support using custom LUI code, and the doesn't contain any decompiled lua files in the tools either. This makes the whole process of using them in a mod a lot more confusing, and because so, requires a few more tools.
DECOMPILING THE LUI
Firstly, to allow the mod tools to compile files with LUI in, I needed to use a mod called L3akMod. This allows the tools to read and compile LUI files, so the game can use them as overrides. The way the game handles mod files, if a copy of a file exists in your mod and in game, it uses and runs the file from the mod. Because of this, you are required to extract the LUI files from the game itself. Using a Call of Duty asset extraction tool made by Scobalula called "Greyhound", it is possible to extract the compiled LUI files from the game and then use a decompiler made by another user in the community to enable them to be read/edited. Obviously, the decompiled scripts are missing 99% of the function and variable names, so it took a lot of effort to reverse engineer what each script was used for. Another issue with using LUI in mods is that when there is an error, the game gives you no information about where or what the error is, apart from just crashing. This makes it a lot harder to debug, so writing small amounts of code incrementally is beneficial.
Using all of this, I was able to repurpose the menu files for the start menu, and reworked all the toggle/slider/dropdown classes to make them use my variable saving functions as opposed to the game. The final result is a menu that feels as if it is part of the base game.
